Când să folosești fiecare pattern

A. Classic Table

Admin dashboards, user management, AI usage

  • Many columns with numeric data
  • Need to compare values across rows
  • Sorting is important
  • Footer totals needed

B. Minimal Table

Case lists, document lists, simple data

  • Fewer columns (3-5)
  • Want cleaner aesthetic
  • Row has title + description
  • Status/badges per row

C. Grouped Table

Time entries, activity logs, tasks

  • Data naturally groups by date/status
  • Users want to collapse old items
  • Per-group subtotals useful
  • Recent items most important

D. List Style

People lists, search results, mobile

  • Avatar/icon important
  • Varied content per row
  • Mobile-first design
  • Click goes to detail page

E. Expandable Rows

Case overview, project summary

  • Overview + details pattern
  • Don't want to navigate away
  • Details are contextual
  • Few rows typically expanded

F. Report with Filters

Billing reports, analytics, exports

  • Dedicated report page
  • Export functionality
  • Multiple filter options
  • Date range selection
