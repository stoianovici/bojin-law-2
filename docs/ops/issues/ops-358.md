# OPS-358: Case Detail Page

**Type**: Feature | **Priority**: P2-Medium | **Platform**: Desktop | **Status**: Verifying
**Parent**: [OPS-338](ops-338.md) | **Phase**: 4 (Pages)
**Blocked by**: OPS-349, OPS-350, OPS-352

## Problem

Case detail needs tabbed interface with 5 tabs: Details, Documents, Time, Messages, Activity.

## Approach

- Create CaseDetailPage with breadcrumb, header, tab bar
- Implement each tab as separate component
- Use existing patterns (tables, lists, cards)

## Done When

- [x] Breadcrumb (Cazuri > Case Title)
- [x] Case header (title, case#, status, client)
- [x] Tab bar (Detalii, Documente, Mape, Sarcini, Comunicări, Timp, Notițe, AI)
- [x] CaseDetailsTab: two-column info layout (existing OverviewTab)
- [x] CaseDocumentsTab: status toggle, view toggle, document grid/table (existing CaseDocumentsList)
- [x] CaseTimeEntriesTab: time entries tab (existing TimeEntriesTab)
- [x] CaseMessagesTab: communications tab (existing CommunicationsTab)
- [x] CaseActivityTab: AI intelligence tab (existing IntelligenceTab)
