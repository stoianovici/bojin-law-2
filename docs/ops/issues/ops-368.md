# OPS-368: Loading/Empty/Error States

**Type**: Feature | **Priority**: P2-Medium | **Platform**: Desktop | **Status**: Verifying
**Parent**: [OPS-338](ops-338.md) | **Phase**: 7 (Polish)
**Blocked by**: OPS-347

## Problem

Need consistent skeleton, spinner, empty state, error state components across all pages.

## Approach

- Create Skeleton components (text, card, table row) with shimmer
- Create Spinner with sizes and optional label
- Create EmptyState with icon, title, description, CTA
- Create ErrorState with retry button
- Add page-specific empty states

## Done When

- [x] Skeleton: text, card, table row with shimmer animation
- [x] Spinner: sm (16px), md (24px), lg (32px), optional label
- [x] EmptyState: icon slot, title, description, CTA button
- [x] ErrorState: error icon, message, retry button
- [x] ErrorBoundary component for React error catching
- [x] Page-specific empty states (no cases, no tasks, etc.) - via presets
- [ ] Skeleton states added to dashboard widgets, case list, task list, documents
