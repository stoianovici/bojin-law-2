# OPS-348: Status Indicator Components

**Type**: Feature | **Priority**: P2-Medium | **Platform**: Desktop | **Status**: Verifying
**Parent**: [OPS-338](ops-338.md) | **Phase**: 1 (Foundation)

## Problem

Status indicators scattered across components. Need unified StatusDot, PriorityBadge, StatusBadge components.

## Approach

- Create StatusDot with color variants and optional glow/pulse
- Create PriorityBadge for urgent/high/medium/low
- Create generic StatusBadge for success/warning/error/info

## Done When

- [x] StatusDot: green/yellow/red colors, optional glow, optional pulse
- [x] PriorityBadge: urgent (red), high (orange), medium (blue), low (gray)
- [x] StatusBadge: generic pill with color variants
- [x] PriorityBar: vertical 3px bars for task priority (urgent/high/medium/low)
- [x] WorkflowStatusBadge: dot + text for workflow statuses (planned/in-progress/review/completed)
- [x] FileTypeBadge: colored badge for document types (PDF/DOC/XLS/IMG)
- [x] CountBadge: numeric badge for counts/notifications
