# OPS-295: Mobile Layout Padding Removal

**Type**: Feature | **Priority**: P1-High | **Complexity**: S

## Problem

MainLayout has `pl-16` padding to account for collapsed sidebar, but on mobile the sidebar will be off-screen (OPS-294), wasting 64px of horizontal space.

## Solution

Change `pl-16` to `md:pl-16` so padding only applies on desktop:

- Mobile: Full width content (no left padding)
- Desktop: Keep 64px left padding for collapsed sidebar

## Implementation

```tsx
// MainLayout.tsx - Update main content div
<div className="flex flex-col h-screen md:pl-16 overflow-hidden">
```

Single line change - add `md:` prefix to `pl-16`.

## Done When

- [ ] Main content uses full width on mobile (390px available, not 326px)
- [ ] 64px left padding preserved on desktop (md+)
- [ ] No horizontal scroll issues
- [ ] Content doesn't shift when sidebar drawer opens on mobile
