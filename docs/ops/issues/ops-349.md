# OPS-349: Navigation Components

**Type**: Feature | **Priority**: P2-Medium | **Platform**: Desktop | **Status**: Verifying
**Parent**: [OPS-338](ops-338.md) | **Phase**: 2 (Composite)
**Blocked by**: Phase 1 (OPS-345, OPS-346, OPS-347, OPS-348)

## Problem

Need reusable navigation patterns: status toggles, view toggles, tab bars, breadcrumbs, filter pills.

## Approach

- Create StatusToggle (button group for status filtering)
- Create ViewToggle (icon-only grid/list toggle)
- Create TabBar with underline indicator and badge support
- Create Breadcrumb component
- Create FilterPills (horizontal scrollable, removable)

## Implementation Notes

### StatusToggle

- Generic typed component supporting any string literal value
- Both controlled (`StatusToggle`) and composable (`StatusToggleGroup`, `StatusToggleItem`) APIs
- Count badges using `CountBadge` from StatusDot
- Two sizes: sm and md

### ViewToggle

- Built-in icons for common views (grid, list, kanban, calendar, table)
- Accepts custom React nodes as icons
- Icon-only with tooltip labels for accessibility

### TabBar

- Underline indicator on active tab
- Badge support for counts
- Horizontal scroll with hidden scrollbar
- Both controlled (`TabBar`) and composable (`TabList`, `Tab`, `TabPanel`) APIs

### Breadcrumb

- Uses Next.js `Link` for navigation
- Current item marked with `aria-current="page"`
- Both simple array API and composable primitives

### FilterPills

- Toggle pills between active/inactive states
- Removable pills with X button
- `ActiveFiltersBar` component for showing active filters with "clear all"
- Horizontal scroll with hidden scrollbar
