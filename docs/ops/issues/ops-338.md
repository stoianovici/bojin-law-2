# OPS-338: Linear UI Adaptation Phase 2 (Epic)

**Type**: Epic | **Priority**: P2-Medium | **Platform**: Both | **Status**: Open

## Overview

Complete the Linear design migration by systematically auditing all remaining pages and components, creating visual mockups, and generating a data-driven implementation plan.

**Parent of**:

- OPS-339: Shared Component Audit (Card, Modal)
- OPS-340: Page Audit - Desktop Styling Consistency
- OPS-341: Page Audit - Mobile Responsiveness
- OPS-342: HTML Mockups & Consolidation
- OPS-343: Generate Implementation Plan

## Background

The initial Linear design migration (OPS-331 through OPS-337) established the core design system:

- Linear tokens in Tailwind config
- Dark mode as default
- Core layout components (Sidebar, TopBar, PageLayout)
- Dashboard widgets
- Task management components

However, approximately **15 pages** and **2 shared components** still use inconsistent styling:

- Hardcoded gray classes (`bg-gray-50`, `text-gray-900`, etc.)
- Heavy shadows on Card/Modal components
- Missing mobile responsiveness on `/cases/new` and `/time-tracking`

## Approach

**Two-phase epic**:

### Phase 1: Discovery (OPS-339 to OPS-341 - Parallel)

Systematic audit of all remaining work, producing documented findings.

### Phase 2: Consolidation (OPS-342 to OPS-343 - Sequential)

Create visual mockups and generate implementation plan.

### Phase 3: Implementation (Issues TBD)

Create targeted issues based on discovery findings. **Not pre-defined** - will be data-driven.
