# OPS-341: Page Audit - Mobile Responsiveness

**Type**: Discovery | **Priority**: P2-Medium | **Platform**: Mobile | **Status**: Complete
**Parent**: [OPS-338](ops-338.md) | **Blocked by**: None | **Parallel**: Yes

## Problem

Two pages were identified as lacking mobile responsiveness:

- `/cases/new` - Complex form layout breaks on mobile
- `/time-tracking` - 3-column grid doesn't adapt

These pages don't have dedicated mobile components and rely on responsive Tailwind classes that may be insufficient.

## Scope

### Primary Pages

| Page             | File                                      | Known Issues                        |
| ---------------- | ----------------------------------------- | ----------------------------------- |
| `/cases/new`     | `apps/web/src/app/cases/new/page.tsx`     | Complex form, no mobile fallback    |
| `/time-tracking` | `apps/web/src/app/time-tracking/page.tsx` | 3-column layout, no mobile fallback |

### Components to Audit

For `/time-tracking`:

- `SummaryView` - `md:grid-cols-4` layout
- `NaturalLanguageEntry` - Input sizing
- `TasksInProgress` - Three-column layout

## Approach

For each page:

1. Test at mobile breakpoints (390px width - iPhone 14)
2. Document layout issues
3. Check touch target sizes (minimum 44x44px)
4. Identify missing responsive utilities
5. Determine if mobile fallback component is needed
