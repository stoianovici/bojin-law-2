# OPS-340: Page Audit - Desktop Styling Consistency

**Type**: Discovery | **Priority**: P2-Medium | **Platform**: Desktop | **Status**: Complete
**Parent**: [OPS-338](ops-338.md) | **Blocked by**: None | **Parallel**: Yes

## Problem

Approximately 15 pages still use hardcoded gray classes instead of Linear design tokens. These pages have inconsistent styling compared to the migrated components.

## Scope

### High Priority Pages (from initial audit)

| Page                  | File                                           | Known Issues                                      |
| --------------------- | ---------------------------------------------- | ------------------------------------------------- |
| `/403`                | `apps/web/src/app/403/page.tsx`                | `bg-gray-50`, `text-gray-900`, `bg-blue-600`      |
| `/cases/new`          | `apps/web/src/app/cases/new/page.tsx`          | Heavy `gray-*` usage, local Card with `shadow-sm` |
| `/admin/users/active` | `apps/web/src/app/admin/users/active/page.tsx` | `gray-*` throughout, hardcoded shadows            |
| `/emails`             | `apps/web/src/app/emails/page.tsx`             | Mixed `gray-*` with dark mode classes             |

### Additional Pages to Audit

- `/reports`
- `/reviews/*`
- `/analytics/*` (tab pages)
- `/admin/performance`
- `/admin/ai-ops/*`
- `/admin/users/pending`
- `/activitate-echipa`

## Approach

For each page:

1. Identify all hardcoded color classes (`gray-*`, `blue-*`, etc.)
2. Document specific lines that need changes
3. Map to appropriate Linear tokens
4. Assess complexity (Simple/Medium/Complex)
