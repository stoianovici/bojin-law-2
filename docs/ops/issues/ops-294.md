# OPS-294: Mobile Sidebar CSS Transform

**Type**: Feature | **Priority**: P1-High | **Complexity**: S

## Problem

Sidebar is always visible on mobile, consuming 64px of screen width. On a 390px screen, this leaves only 326px for content - making the UI cramped and difficult to use.

## Solution

Add CSS transforms to hide sidebar off-screen by default on mobile:

- Add `-translate-x-full md:translate-x-0` to slide sidebar off-screen on mobile
- When `isSidebarCollapsed` is false (drawer open), add `translate-x-0` to show
- Keep existing expanded width (`w-64`) for mobile drawer view
- Backdrop overlay already exists, adjust to show on mobile drawer open

## Implementation

```tsx
// Sidebar.tsx - Update aside className
<aside
  className={`
    ${className}
    flex flex-col
    bg-white border-r border-gray-200
    transition-all duration-300 ease-in-out
    w-64 md:${isSidebarCollapsed ? 'w-16' : 'w-64'}
    h-screen fixed top-0 left-0 z-[60]
    overflow-y-auto
    // Mobile: hidden by default, shown when !isSidebarCollapsed
    ${isSidebarCollapsed ? '-translate-x-full' : 'translate-x-0'}
    md:translate-x-0
  `}
>
```

Key changes:

1. Mobile always uses `w-64` (full drawer)
2. Desktop uses conditional `w-16`/`w-64`
3. Mobile uses translate to show/hide
4. Desktop always `translate-x-0` (visible)
