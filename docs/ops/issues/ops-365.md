# OPS-365: Clients & Billing Pages

**Type**: Feature | **Priority**: P2-Medium | **Platform**: Desktop | **Status**: Verifying
**Parent**: [OPS-338](ops-338.md) | **Phase**: 6 (Admin & Settings)
**Blocked by**: OPS-352, OPS-353

## Problem

Clients and Billing pages need minimal tables with CRUD modals.

## Approach

- Create ClientsListPage with search, status filter
- Create NewClientModal, ClientDetailPage
- Create BillingPage with invoice table
- Create NewInvoiceModal, InvoiceDetailPage

## Done When

- [x] ClientsListPage: search, status filter, minimal table
- [x] NewClientModal: name, contact, email, phone, address
- [x] ClientDetailPage: info, cases list, billing history
- [x] BillingPage: period filter, status toggle, invoice table
- [x] NewInvoiceModal: client, case, line items, totals
- [x] InvoiceDetailPage: header, line items, actions (mark paid, send, download)
