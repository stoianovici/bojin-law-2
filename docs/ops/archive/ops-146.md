# OPS-146: List Animation Integration

**Type**: Feature
**Priority**: P2-Medium
**Status**: Closed
**Complexity**: M
**Parallel**: Blocked by OPS-144

## Problem

Email lists, document lists, and extracted items appear/disappear instantly. No visual feedback when items are added or removed, making the UI feel static and unresponsive.

## Solution

Apply the motion components from OPS-144 to high-traffic list components:

- `ThreadList` - email thread items
- `DocumentList` - document grid/list items
- `ExtractedItemsSidebar` - AI-extracted action items

## Implementation

### ThreadList

```tsx
import { AnimatedListItem } from '@/components/motion';
import { AnimatePresence } from 'framer-motion';

<AnimatePresence mode="popLayout">
  {threads.map((thread) => (
    <AnimatedListItem key={thread.id} layoutId={thread.id}>
      {/* existing thread item content */}
    </AnimatedListItem>
  ))}
</AnimatePresence>;
```

### DocumentList

```tsx
import { StaggerChildren, StaggerItem } from '@/components/motion';

<StaggerChildren className="grid gap-4">
  {documents.map((doc) => (
    <StaggerItem key={doc.id}>{/* existing document card */}</StaggerItem>
  ))}
</StaggerChildren>;
```

### ExtractedItemsSidebar

```tsx
import { AnimatedListItem } from '@/components/motion';
import { AnimatePresence } from 'framer-motion';

<AnimatePresence>
  {items.map((item) => (
    <AnimatedListItem key={item.id}>{/* existing item content */}</AnimatedListItem>
  ))}
</AnimatePresence>;
```
