# OPS-145: Collapsible Height Animation Pattern

**Type**: Feature
**Priority**: P2-Medium
**Status**: Closed
**Complexity**: S
**Resolved**: 2024-12-24

## Problem

Collapsible sections (MorningBriefing, AIDraftResponsePanel, sidebars) expand/collapse instantly without smooth height transition. This feels jarring compared to modern UI expectations.

## Solution

Update `ui/collapsible.tsx` to use Radix data attributes with CSS animations for smooth height transitions.

## Implementation

### Update Collapsible Component

```tsx
// apps/web/src/components/ui/collapsible.tsx
'use client';

import * as CollapsiblePrimitive from '@radix-ui/react-collapsible';
import { forwardRef } from 'react';
import { clsx } from 'clsx';

const Collapsible = CollapsiblePrimitive.Root;
const CollapsibleTrigger = CollapsiblePrimitive.Trigger;

const CollapsibleContent = forwardRef<
  React.ElementRef<typeof CollapsiblePrimitive.Content>,
  React.ComponentPropsWithoutRef<typeof CollapsiblePrimitive.Content>
>(({ className, children, ...props }, ref) => (
  <CollapsiblePrimitive.Content
    ref={ref}
    className={clsx(
      'overflow-hidden',
      'data-[state=open]:animate-expandHeight',
      'data-[state=closed]:animate-collapseHeight',
      className
    )}
    {...props}
  >
    {children}
  </CollapsiblePrimitive.Content>
));
CollapsibleContent.displayName = 'CollapsibleContent';

export { Collapsible, CollapsibleTrigger, CollapsibleContent };
```

## Resolution

Updated `collapsible.tsx` to wrap `CollapsibleContent` with animation classes using Radix data attributes:

- `data-[state=open]:animate-expandHeight` for smooth expand
- `data-[state=closed]:animate-collapseHeight` for smooth collapse
- Added `overflow-hidden` to prevent content clipping during animation

Used in 7 components: MorningBriefing, SuggestionCard, ResponseSuggestionPanel, ChangeSummaryPanel, AIConcernsPanel, DeadlineWarningBanner, CompletenessIndicator.
