# OPS-147: Card Hover & Loading State Polish

**Type**: Feature
**Priority**: P2-Medium
**Status**: Closed
**Complexity**: S
**Parallel**: Blocked by OPS-143

## Problem

Cards and list items have minimal hover feedback (shadow only). Loading overlays appear instantly without fade, creating jarring visual transitions.

## Solution

1. Enhance `CaseCard` hover with subtle scale + shadow elevation
2. Add `animate-fadeIn` to loading overlays
3. Apply consistent hover pattern to table rows

## Implementation

### CaseCard Hover Enhancement

```tsx
<div
  className={clsx(
    'rounded-lg border bg-white p-4',
    'transition-all duration-200',
    'hover:shadow-lg hover:scale-[1.01] hover:-translate-y-0.5',
    'cursor-pointer'
  )}
>
```

### DocumentPreviewModal Loading Overlay

```tsx
{
  loading && (
    <div className="absolute inset-0 flex items-center justify-center bg-white/80 z-10 animate-fadeIn">
      <Loader2 className="h-8 w-8 animate-spin text-blue-600" />
    </div>
  );
}
```

### CaseListTable Row Hover

```tsx
<tr className="transition-colors duration-150 hover:bg-gray-50 cursor-pointer">
```
