## Description

The application has mixed English and Romanian UI strings throughout the codebase. Approximately 30-40% of UI text is still in English, scattered across ~657 TypeScript/TSX files. There is no existing i18n/translation system in place - all strings are hardcoded in components.

## Scope Summary

- **Estimated English strings to translate:** 300-500+
- **Files affected:** 80-250 primary UI files, plus test files
- **No i18n library installed** (next-intl, i18next, react-intl not present)
- **Existing Romanian translations:** ~60-70% of UI, well-done

## Key Areas Requiring Translation

### High Priority

1. **Navigation (Sidebar.tsx)**
   - "Dashboard" → "Tablou de Bord"
   - "Analytics" → "Analiză"
   - "My Cases" → "Cazurile Mele"

2. **Admin Pages**
   - "Discovery Dashboard" → "Tablou de Descoperire"
   - "Performance Dashboard" → "Tablou de Performanță"

3. **Task Management (46 files)**
   - "Calendar", "Kanban" tab labels
   - "Create new task" → "Creează sarcină nouă"

4. **Error Pages**
   - "Return to Dashboard" → "Înapoi la Tablou"

### Medium Priority

- Case management components (66 files)
- Document editor components (18 files)
- Dashboard components (24 files)
- Communication components (36 files)

### Low Priority

- Test files (for consistency)
- Storybook story files

## Root Cause

Incremental development without a formal i18n strategy. Romanian strings were added directly to components, but some sections (especially navigation and admin areas) retained English text.

## Recommended Approach

**Option A: Direct string replacement (Quick fix)**

- Replace English strings directly with Romanian in components
- Faster but no multi-language support

**Option B: Implement next-intl (Recommended for scalability)**

- Install next-intl library
- Create translation JSON files
- Update components to use translation hooks
- Enables future multi-language support
