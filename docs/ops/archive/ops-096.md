# OPS-096: Task Card Display Redesign

## Summary

Redesign task cards to prominently show title and case name instead of task type. Type indicated via subtle color coding.

## Background

Current task cards show:

- Time badge (if set)
- Task title
- Task type badge (Research, Meeting, etc.)

Users care more about "which case" than "what type" - the case context is more actionable. Type can often be inferred from the title.

## Requirements

### Acceptance Criteria

1. Task cards show: title (prominent) + case name (secondary)
2. Task type indicated by left border color or subtle icon
3. Priority still shown via border thickness
4. Time badge still shown for time-specific tasks
5. Applies to Calendar, Kanban, and List views

### Visual Design

```
Current:
┌─────────────────────────┐
│ [14:00] Task Title      │
│ [Research]              │  ← Type badge
└─────────────────────────┘

Proposed:
┌─────────────────────────┐
│ ▌Task Title             │  ← Colored left border = type
│   Popescu v. SRL        │  ← Case name (gray, smaller)
└─────────────────────────┘

With time:
┌─────────────────────────┐
│ ▌14:00 Task Title       │
│   Popescu v. SRL        │
└─────────────────────────┘
```

### Color Legend (existing)

- Research: Blue (#3B82F6)
- DocumentCreation: Green (#10B981)
- DocumentRetrieval: Purple (#8B5CF6)
- CourtDate: Red (#EF4444)
- Meeting: Amber (#F59E0B)
- BusinessTrip: Indigo (#6366F1)

## Implementation

### Files to Modify

1. **MultiWeekCalendarView.tsx** - TaskCard subcomponent
2. **KanbanBoard.tsx** - KanbanTaskCard subcomponent
3. **ListView.tsx** - Table row display
4. **useTasks.ts** or GraphQL query - Include case title in task fetch

### GraphQL Changes

```graphql
# Ensure task query includes case
type Task {
  # ... existing fields
  case {
    id
    title
  }
}
```

### Accessibility

- Color-only type indication needs supplementary indicator (tooltip on hover, icon)
- Ensure sufficient contrast for all type colors
