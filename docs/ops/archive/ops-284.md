# OPS-284: Frontend Draft Messaging

**Type**: Feature
**Priority**: P1-High
**Status**: Closed
**Blocked By**: ~~OPS-280~~ (Complete)

## Problem

Frontend UI still shows "Trimite" (Send) messaging when all outgoing emails now go to Outlook Drafts folder. Need to update all send-related UI text to reflect draft mode.

## Solution

Update all email sending UI components to use draft-focused messaging:

- "Trimite" → "Salvează în Ciorne"
- "Trimis!" → "Salvat în Ciorne!"
- Confirmation dialogs mention Outlook Drafts folder

## Implementation

### Components Updated

1. **EmailComposer.tsx**
   - Send button: "Salvează în Ciorne" (line 385)
   - Confirmation dialog: "Salvează în Ciorne Outlook?" (line 405)
   - Dialog text explains user needs to open Outlook to review and send

2. **ComposeInterface.tsx**
   - Send button: "Salvează în Ciorne" (line 722)
   - Success message: "Salvat în Ciorne!" (line 712)
   - Loading state: "Se salvează..." (line 717)

3. **DraftIndicator.tsx**
   - "Sent" status label: "În Ciorne" (line 50)
   - Blue styling to indicate draft state
